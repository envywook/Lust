# Security Policy

## Supported versions

Lust is currently an early alpha. Only the latest commit on the default branch is considered for security fixes.

## Reporting a vulnerability

Do not open a public issue containing subscription URLs, proxy credentials, private keys, or exploitable details.

Use GitHub's private vulnerability reporting feature when enabled for the repository. Include:

- affected commit/version;
- reproduction steps;
- security impact;
- suggested mitigation, if known.

Never include live production credentials. Revoke any credential accidentally disclosed in an issue, log, screenshot, or build artifact.
